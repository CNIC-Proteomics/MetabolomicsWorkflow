TableMerger

TableMerger receives two user-specified tables as input and merge them in a single matrix. The equivalent features of both tables are identified thanks to the experimental mass (Experimental mass) and the retention time (RT [min]). However, if the RT is not contained in any of the input tables, only the experimental mass will be used. The output of this module will be a single matrix with the combined information from the two data sets. Since the experimental mass of the features can be expressed with different number of decimals in both tables, the user will be able to specify the number of decimal places to which to round.
Parameters
  • N_Digits: Number of decimal places to which the experimental mass of the features will be rounded to make the "merge".
  • OutputName: Name of the output file. By default, the name of the output file will be that of the input file with prefix "TableMerged_"
  • OutputColumns: Name of the columns present in the output table. By default, they will all appear.
